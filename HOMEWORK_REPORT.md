# 🤖 Проверка домашки ✨

Статический проект-шаблон с React-приложением. Выводит «Hello, World!» на странице. Используется React 19, Less для стилей, Vite 8 для сборки.

## 1) ✅✅✅✅ Настройка репозитория

- ✅ Основная ветка — master
- ✅ Настроена подпись коммитов
- ✅ Если используется НЕ npm, присутствует поле `packageManager` в `package.json` и другие необходимые файлы и папки (yarnrc для старших версий yarn и т.п.)
- ✅ После `npm install` / `yarn install` / etc. не появляется дифф.

ℹ️ Используется npm (есть package-lock.json).

## 2) ✅ Editorconfig

- ✅ Есть `.editorconfig` в корне проекта

ℹ️ Настроены: utf-8, lf, final newline, trim trailing whitespace, indent 4 spaces (json — 2 spaces).

## 3) ✅✅✅✅ TypeScript (опционально)

- ✅ Конфиг: `tsconfig.json`
- ✅ Есть npm-скрипт для проверки типов (`tsc --noEmit`)
- ✅ Ошибка типов ловится запуском скрипта проверки типов
- ✅ Скрипт проверки типов проходит без ошибок

ℹ️ Настроены strict mode, path aliases (`@/*` → `./src/*`), composite project с tsconfig.node.json.

💡 Используется конфиг с алиасами для импортов.

## 4) ✅✅✅✅✅✅✅ Линтер

- ✅ Зависимости: ESLint, Biome или альтернатива
- ✅ Конфиг: `.eslintrc.*`, `biome.json` или аналог
- ✅ npm-скрипт для проверки конкретных файлов (npm run SCRIPTNAME file1 file2 ...) обрабатывает только переданные файлы
- ✅ npm-скрипт для проверки всего проекта (npm run SCRIPTNAME) обрабатывает все подходящие файлы
- ✅ В одном из npm-скриптов используется автоисправление (флаг `--fix` или аналог)
- ✅ Запуск линтера на всём проекте без автофикса проходит без ошибок
- ✅ При наличии ошибки в JS линтер не проходит

ℹ️ Скрипты: `lint` (с --fix), `lint-check` (только проверка), `lint-files` / `lint-files-check` (для конкретных файлов).

💡 Используется конфиг @hh.ru.

## 5) ✅✅✅✅✅✅✅✅ Форматтер

- ✅ Зависимости: Prettier, Biome или альтернатива
- ✅ Форматируется JS, CSS, HTML, JSON
- ✅ В форматтер не попадают файлы, которые он не умеет форматировать (ограничены маской ЛИБО используется `--ignore-unknown` или аналог)
- ✅ npm-скрипт для проверки/форматирования конкретных файлов (npm run SCRIPTNAME file1 file2 ...) обрабатывает только переданные файлы
- ✅ npm-скрипт для проверки/форматирования всего проекта (npm run SCRIPTNAME) обрабатывает все подходящие файлы
- ✅ В одном из npm-скриптов используется автоисправление (флаг `--write` или аналог)
- ✅ Запуск проверки на всём проекте без автофикса проходит без ошибок
- ✅ При неправильном форматировании в JS проверка без автофикса не проходит

ℹ️ Скрипты: `format` (с --write), `format-check` (только проверка), `format-files` / `format-files-check` (для конкретных файлов). Используется `--ignore-unknown`.

## 6) ✅✅✅✅✅✅ Pre-commit проверки

- ✅ При `npm install` устанавливается pre-commit хук (без запуска скриптов вручную)
- ✅ Установлен lint-staged или аналог
- ✅ В конфиге lint-staged используются только npm-скрипты для консистентности (пакеты не вызываются напрямую)
- ✅ При попытке закоммитить код с исправляемой ошибкой она исправляется автофиксом и коммит проходит
- ✅ При попытке закоммитить код с НЕисправляемой ошибкой коммит НЕ проходит
- ✅ При попытке закоммитить неправильно отформатированный код он форматируется и коммит проходит с отформатированным кодом

ℹ️ Хук копируется из `tools/githooks/` в `.git/hooks/` через `postinstall` скрипт. В хуке запускается `npx lint-staged`.

## 7) ✅✅✅ React JSX транспиляция

- ✅ Есть React в зависимостях (версия не ниже 18)
- ✅ Есть JSX файлы в проекте
- ✅ Настроена обработка JSX (Babel, Webpack, Vite, etc.)

## 8) ✅✅ Зависимости

- ✅ Зависимости правильно распределены по dependencies и devDependencies
- ✅ Используется хотя бы один пакет NPM-пакет из dependencies, кроме react и react-dom

ℹ️ В dependencies: `modern-normalize`. В devDependencies: eslint, prettier, typescript, vite и др.

💡 Настроена установка точных версий в `.npmrc` (`save-exact=true`).

## 9) ✅✅✅ Настроена сборка проекта

- ✅ Есть npm-скрипт `build` или аналогичный
- ✅ Проект собирается без ошибок
- ✅ Папка, в которой собирается проект, в `.gitignore`

ℹ️ Сборка в папку `build` (настроено в vite.config.ts).

💡 Настроены алиасы для импортов (`@/*` → `./src/*`).

💡 Есть комбинированный скрипт `check` для запуска всех проверок.

## 10) ✅✅ Настроен dev-сервер

- ✅ Есть npm-скрипт `dev`, `start` или аналогичный
- ✅ Сервер запускается без ошибок

## 11) ✅✅✅ Деплой в GitHub Pages npm-скриптом

- ✅ Есть зависимость `gh-pages`
- ✅ Есть npm-скрипт `deploy` или аналогичный, использующий `gh-pages`
- ✅ Настроен относительный путь для продакшн-сборки (например, поле `base` в `vite.config.js`)

## 12) ✅✅✅✅ GitHub Action для деплоя изменений из мастера

- ✅ Есть workflow-файл в `.github/workflows/`
- ✅ Воркфлоу триггерится на push в master
- ✅ Перед сборкой прогоняются проверки с помощью npm-скриптов
- ✅ Код собирается и деплоится в GitHub Pages

ℹ️ Используются actions/upload-pages-artifact@v4 и actions/deploy-pages@v4. Перед билдом запускается `npm run check` (ts-check + lint-check + format-check).
